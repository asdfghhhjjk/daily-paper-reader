## 摘要
传统的组织图像分析软件为细胞分析提供了基础功能，包括分割、基本形态特征提取和空间组织分析。然而，这些工具通常需要人工干预，且未能与代码驱动的自动化良好集成，限制了复杂空间组织研究的效率和可扩展性。此外，它们对定制化分析提供的灵活性有限，通常仅支持一组固定的预实现空间细胞特征。为解决这些局限性，我们提出了CodeCytos，一个基于编码的推理代理框架，能够实现对空间分子成像数据的动态、可编程交互，以提高自动化和定制化能力。CodeCytos旨在简化自定义空间细胞特征的探索，并适应多样化的研究需求。我们通过四个由专家策划的不同组织类型数据集（额叶皮层、非小细胞肺癌、胰腺和扁桃体）的案例研究展示了其实用性。我们在现实的最小提示设置下评估了CodeCytos，其中生物科学家提出简单问题，无需任务特定指令或空间细胞分析的上下文信息，并基准测试了多个具有强编码能力的LLM骨干。我们进一步证明，引入定制的、领域无关的小样本上下文编码推理示例（在空间分析领域外随机采样的演示）可以显著提升性能，而无需昂贵且需要专家精心制作的领域内演示。总体而言，CodeCytos优于基线方法，凸显了代码动作代理在辅助空间分子成像中自定义特征探索和加速生物标志物发现方面的潜力。

## Abstract
Conventional tissue image analysis software provides foundational capabilities for cellular analysis, including segmentation, basic morphological feature extraction, and spatial organization analysis. However, these tools often require manual intervention and are not well integrated with code-driven automation, limiting efficiency and scalability for complex spatial tissue studies. In addition, they offer limited flexibility for custom analyses, as they typically support only a fixed set of pre-implemented spatial cellular features. To address these limitations, we propose CodeCytos, a coding-based reasoning agent framework that enables dynamic, programmable interaction with spatial molecular imaging data to improve automation and customization. CodeCytos is designed to streamline the exploration of custom spatial cellular features and adapt to diverse research needs. We demonstrate its utility through case studies on four expert-curated datasets from distinct tissue types: frontal cortex, non-small-cell lung cancer, pancreas, and tonsil. We evaluate CodeCytos under a realistic minimal prompt setting, where bioscientists pose simple questions without task-specific instructions or contextual information about spatial cellular analysis, and benchmark multiple LLM backbones with strong coding capabilities. We further show that incorporating tailored, domain-agnostic few-shot in-context coding-reasoning examples (randomly sampled demonstrations outside the spatial analysis domain) can substantially improve performance without requiring costly, expert-crafted in-domain demonstrations. Overall, CodeCytos outperforms baseline approaches, highlighting the potential of code-action agents to assist with custom feature exploration in spatial molecular imaging and to accelerate biomarker discovery.

### 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

- **核心思想**：将大型语言模型（LLM）的预训练编程知识与 **ReAct（Reasoning + Acting）** 框架结合，并采用 **CodeAct** 范式，将代理的动作空间统一为可执行的 Python 代码。代理通过“思考（Thought）→ 代码生成（Code）→ 执行观察（Observation）”的循环迭代，自主完成从问题理解到最终特征提取的全过程。
- **关键技术细节**：
  - **ReAct 扩展**：动作空间 `A` 被扩展为 `A ∪ L`，其中 `L` 为自然语言“思考”动作，用于组织信息、制定计划、自我纠错，但不直接影响环境。
  - **CodeAct 范式**：环境动作 action 以可执行 Python 代码形式输出，代码执行结果（输出、日志、错误）作为新观察加入上下文。代码支持数据流、控制流和库调用，利用 LLM 的编程先验，减少对人工示范的依赖。
  - **多轮执行与自纠正**：每个代码动作在沙箱中执行，若失败则利用错误信息自动修正代码并重执行，直到成功或达到最大步数（本文设为10步）。
  - **少样本编程-推理示例（Few-shot Coding-Reasoning Exemplars）**：向系统提示中预置少量**领域无关**（非空间分析）的编码+推理轨迹示例，用于规范代理的交互结构（`<thought>`、`<code>`、`<obs>` 标签），提升代理对预期行为的遵循程度。这些示例不需要专家标注，成本低。
- **算法流程**：
  1. 接收用户文本查询（无额外任务指令或上下文）。
  2. 迭代执行：
     - **Thought**：基于当前上下文分析目标，规划下一步。
     - **Act（CodeAct）**：生成 Python 代码，调用细胞分割/分类工具或其他库。
     - **Observe**：执行代码，捕获输出/错误，更新上下文。
  3. 当生成最终答案或达到步数上限时终止。

### 6. 论文的主要结论与发现

1. **CodeCytos（CodeAct 基）显著优于传统工具增强 LLM**：在几乎所有模型和数据集上，CodeAct 设置（即使零样本）都远高于工具增强 LLM 设置（包括少样本 CoT）。
2. **领域无关的少样本编码-推理示例大幅提升性能**：对于开源 LLM，添加几个非空间分析领域的随机编码推理示例即可带来巨大提升（例如 Qwen2.5-Coder 在 Frontal Cortex 的 AUP@k 从 0.04 提升至 0.64）。这一提升在 M3ToolEval 上也得到验证（例如 Facebook CWM 成功率从 37.8% 提升至 54.9%）。
3. **最适合的 LLM 骨干存在差异**：Devstral-2-123B 和 GLM-4.5-Air 在总体上表现最强；但一些模型（如 Kimi-Linear）在 CodeAct 设置中提升有限，说明骨干的编码先验和架构会影响效果。
4. **最近邻距离（NND）类任务相对最容易**，而图/拓扑和空间统计类任务更具挑战性，且模型间差异更大。
5. **Pass@k 曲线通常在 k≈10 后饱和**，表明增加尝试次数能提升成功率但无法达到完美，仍需改进代理的可靠性和规划能力。

  - 最小提示设置消除了提示工程偏差。

  - 引入 AUP@k 作为综合指标，比单独 pass@k 更直观。
  - 报告置信区间，统计严谨。
- **资源效率**：不依赖大型专有模型（如 GPT-4），关注开源模型，有望推动社区可复现研究。
- **应用覆盖面广**：覆盖四种不同组织、五种主要特征类别，案例丰富。

### 8. 不足与局限

- **少样本示例并非对每个模型都有效**：部分模型（如 GLM-4.5-Air 在某个任务）添加示例后性能持平甚至略降，说明引导策略仍需针对模型调整。
- **依赖 LLM 的预训练知识**：虽然使用了开源模型，但与顶尖专有模型（如 GPT-4）的性能差距仍然存在（原 CodeAct 论文中最佳零样本开源模型仅 13.4%，本文通过少样本大幅提升但仍未到 100%）。
- **物理标度假设可能不准确**：当图像未提供像素-微米比例时，代理默认 1 px = 1 μm，可能偏离实际，用户需手动核实。
- **仅评估最大 10 步**：对于更复杂、需要更多推理步骤的任务可能受限。
- **细胞分割/分类工具需用户预先选择**：不是完全端到端的全自动系统，仍需要用户指定并上传分割/分类结果。
- **泛化性验证有限**：仅测试了四个组织类型和 CosMx 平台，跨平台、跨染色方式的泛化能力未知。
- **M3ToolEval 上改进幅度较小**，说明领域无关示例对简单工具调用任务帮助有限，主要受益于需要复杂推理的代码生成任务。
- **未讨论推理延迟和计算成本**，部署 100B+ 模型的实时性及大规模使用时的资源需求未被评估。
