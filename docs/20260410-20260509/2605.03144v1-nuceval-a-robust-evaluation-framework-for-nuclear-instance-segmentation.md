## 摘要
在计算病理学中，核实例分割是一项具有许多下游临床应用的基础任务。随着深度学习的兴起，针对该任务已经提出了许多方法，包括卷积神经网络（CNN）和视觉Transformer（ViT），以及基于机器学习和非机器学习的前后处理技术，以进一步提升性能。然而，一个较少受到关注的基本方面是评估流程。在本研究中，我们识别出与核实例分割评估相关的四个关键问题，并提出相应的解决方案。我们提出的改进措施——处理模糊区域、分数归一化、重叠实例以及边界不确定性——被整合进一个名为 NucEval 的统一框架中，使核实例分割的评估更加稳健。我们使用 NuInsSeg 数据集对该评估流程进行了验证，该数据集具有独特特性，使其特别适合本研究；此外，还使用两个额外的外部数据集以及三种基于 CNN 和 ViT 的核实例分割模型，来展示这些改进对实例分割指标的影响。代码以及完整的使用指南与示例均已公开，地址为：https://github.com/masih4/nuc_eval。

## Abstract
In computational pathology, nuclear instance segmentation is a fundamental task with many downstream clinical applications. With the advent of deep learning, many approaches, including convolutional neural networks (CNNs) and vision transformers (ViTs), have been proposed for this task, along with both machine learning-based and non-machine learning-based pre- and post-processing techniques to further boost performance. However, one fundamental aspect that has received less attention is the evaluation pipeline. In this study, we identify four key issues associated with nuclear instance segmentation evaluation and propose corresponding solutions. Our proposed modifications, namely handling vague regions, score normalization, overlapping instances, and border uncertainty, are integrated into a unified framework called NucEval, which enables robust evaluation of nuclear instance segmentation. We evaluate this pipeline using the NuInsSeg dataset, which provides unique characteristics that make it particularly suitable for this study, as well as two additional external datasets, with three CNN- and ViT-based nuclear instance segmentation models, to demonstrate the impact of these modifications on instance segmentation metrics. The code, along with complete guidelines and illustrative examples, is publicly available at: https://github.com/masih4/nuc_eval.

## 二、方法论与核心思想

### 1. 方法总体概念
NucEval 通过在常用分割评估指标（PQ、AJI、Dice、DQ、SQ）上进行四项关键改进，使评估更公正、更鲁棒。这四个模块可独立或联合使用。

### 2. 四项核心修改（Modifications）
1. **模糊区域处理（Handling of vague areas）**
   - 问题：部分图像区域（如染色不佳或核过于密集）无法被准确标注。
   - 方法：利用数据集中“vague mask”，在评估时**剔除这些区域**。  
     - 实例与模糊区的重叠率超过阈值则删除（默认阈值 0.25）。
     - 重叠率阈值可调（参数 `overlap_thresh_amb`）。
   - 效果：避免在不确定区域对模型作出无意义的惩罚。

2. **分数归一化（Score normalization）**
   - 问题：不同测试图像包含不同数量核，导致评分偏向核少的图像。
   - 方法：引入核数量加权平均，使每张图像的权重与其核数量成比例。  
     - 即加权平均公式：
       \[
       \bar{s} = \frac{\sum_i n_i s_i}{\sum_i n_i}
       \]
       其中 \(n_i\) 为第 i 图像核数，\(s_i\) 为其对应指标值。
   - 效果：消除样本密度引起的偏差，提高整体评分的代表性。

3. **重叠实例处理（Overlapping regions）**
   - 问题：标注文件常将重叠区域随意分配给“最后标注的核”，引入排序偏差。
   - 方法：
     - 将重叠区域记入**所有相关实例**；
     - 基于原始 ROI 文件或转换为独立二值掩膜集进行计算。
   - 效果：保持真实的几何信息，减少人工标注顺序带来的不公。

4. **边界不确定性区（Border uncertainty zone）**
   - 问题：人工标注在核边界往往存在几个像素的偏差。
   - 方法：为每个实例设定一个“双边形环形区”：
     - 对实例分别执行膨胀和腐蚀操作；
     - 环区 = 二者差集；
     - 环区中的像素从预测与真值中剔除。
   - 参数：`zone_width` 控制环宽（默认 1 像素）。
   - 效果：减轻微小标注误差带来的评分波动。

### 3. 综合框架与算法实现
- 框架核心函数：`NucEval()`。  
  可选择启用任意组合的四项修改。
- 基于 Python 实现，依赖库：`numpy`, `scipy`, `opencv-python`。
- 自带伪代码与开源示例（GitHub: [masih4/nuc_eval](https://github.com/masih4/nuc_eval)）。

## 六、主要结论与发现

1. **所有修改均可提升评估稳定性与指标均值**；
2. **边界不确定性区 (#4)** 是最稳健且提升最大的一项；
3. **模糊区域处理 (#1)** 和 **分数归一化 (#2)** 增益显著但依数据集依赖；
4. **重叠实例处理 (#3)** 效果最不明显，因重叠区占比小；
5. **综合应用四项修改后：**
   - 三模型在 NuInsSeg 数据集上的 PQ 提升约 10–12%；  
   - 在子集上改进更显著（PQ 提升 38–43%）。
6. 框架适用于任意分割任务（不限病理领域），且已公开代码供验证。

## 八、不足与局限

- **数据依赖性**：模糊区域和重叠区域信息仅少数数据集具备，限制了框架应用范围；
- **标注主观性**：NuInsSeg 中模糊区由单一标注者定义，存在主观差异风险；
- **统一边界区宽度 (`zone_width`) 固定**：未能适应不同核大小与形态，可进一步研究自适应策略；
- **算力规模有限**：仅单机验证，尚未展示大规模分布式或多GPU性能；
- **实验焦点偏向指标层面**：尚缺对临床下游任务绩效提升的验证。
