## 摘要
多实例学习（Multiple Instance Learning, MIL）是计算病理中用于分类千兆像素整张切片图的主要框架。MIL一般遵循以下步骤：（1）提取图块特征；（2）应用线性层以获得任务相关的图块特征；（3）将这些图块聚合为切片特征用于分类。尽管现有研究已在优化图块特征提取和聚合方面投入大量工作，但尚无人关注第二步——该关键层负责将通用特征转换为任务特定特征。我们推测，这一层是被忽视的性能瓶颈，并认为可通过针对每个图块表型的低秩变换实现更强的特征表示，从而与现有的任何MIL方法产生协同效应。为此，我们提出了 MAMMOTH ——一种参数高效的多头专家混合模块，可在几乎不增加参数量的情况下提升任意MIL模型的性能。在八种MIL方法和十九种不同分类任务的实验中，我们发现这种任务特定变换对性能的影响比聚合方法的选择更显著。例如，当应用 MAMMOTH 后，即使是如最大池化或平均池化这样的简单方法，其平均性能也高于使用标准线性层的任意方法。总体而言，MAMMOTH 在152种实验配置中有130种性能提升，平均提升幅度为+3.8%。代码可在 https://github.com/mahmoodlab/mammoth 获取。

## Abstract
Multiple Instance Learning (MIL) is the predominant framework for classifying gigapixel whole-slide images in computational pathology. MIL follows a sequence of 1) extracting patch features, 2) applying a linear layer to obtain task-specific patch features, and 3) aggregating the patches into a slide feature for classification. While substantial efforts have been devoted to optimizing patch feature extraction and aggregation, none have yet addressed the second point, the critical layer which transforms general-purpose features into task-specific features. We hypothesize that this layer constitutes an overlooked performance bottleneck and that stronger representations can be achieved with a low-rank transformation tailored to each patch's phenotype, yielding synergistic effects with any of the existing MIL approaches. To this end, we introduce MAMMOTH, a parameter-efficient, multi-head mixture of experts module designed to improve the performance of any MIL model with minimal alterations to the total number of parameters. Across eight MIL methods and 19 different classification tasks, we find that such task-specific transformation has a larger effect on performance than the choice of aggregation method. For instance, when equipped with MAMMOTH, even simple methods such as max or mean pooling attain higher average performance than any method with the standard linear layer. Overall, MAMMOTH improves performance in 130 of the 152 examined configurations, with an average $+3.8\%$ change in performance. Code is available at https://github.com/mahmoodlab/mammoth.

  - 当前所有补丁都经过同一线性变换，未能捕捉组织形态学的多样性。
  - 单一线性层成为性能瓶颈，导致任务特定表示能力不足。
- **研究目标**：
  - 通过**更灵活、表型感知**的特征变换机制替代原线性层；
  - 在不显著增加参数的情况下提升模型表达力与泛化性。

## 二、方法论：MAMMOTH 模型设计

### 1. 核心思想

- 提出一种**多头小专家混合模块（Mixture of Mini Experts, MAMMOTH）**，可**直接替代任何 MIL 框架中的线性层**。
- 每个 patch 的特征通过多个微型专家（mini experts）进行低秩、表型特定的变换，实现对不同组织形态的专门化处理。
- 模块设计兼顾**参数高效性（parameter efficiency）**与**任务特定性（task specificity）**。

### 2. 模块结构概览（对应图 2）

MAMMOTH 的流程可简述为：

1. **特征输入划分（Multi-head Partitioning）**  
   - 将每个 patch 的高维嵌入划分为多个“头部（heads）”，每个 head 处理不同的特征子空间。  
   - 增强表示的细粒度控制。

2. **基于槽（Slots）的特征汇聚（Slot-based Pooling）**  
   - 每个专家（Expert）设有多个可学习的**槽原型（slot prototypes）**；
   - 通过与输入 patch 特征的相似度，执行 softmax 加权汇聚；
   - 实现**软专家分配（soft expert assignment）**，改善梯度流与专家利用均衡性。

3. **低秩专家变换（Low-rank Experts）**  
   - 每个专家的线性映射矩阵被分解为：
     \[
     W_{\text{full}}^{(k)} = W_{\text{low}}^{(k)} \cdot \Phi
     \]
     其中 \( W_{\text{low}}^{(k)} \) 为专家特定权重，Φ 为共享低秩矩阵。
   - 通过矩阵分解与权重共享，保证参数量基本不变，支持大规模专家扩展。

4. **拼接输出与降维汇聚（Concatenation & Reduction）**  
   - 将所有头和专家输出拼接成紧凑的任务特定嵌入集合；
   - 相比原始输入 patch 数量，特征数量减少约 25 倍，利于稳定聚合。

### 3. 模型特性总结

- **Soft Routing**：避免了稀疏 MoE 的训练不稳定性；
- **多头结构**：应对高维输入（>1024 维）；
- **低秩分解**：提升参数与计算效率；
- **兼容性强**：可无缝嵌入 ABMIL、CLAM、TransMIL、Transformer 等所有常见 MIL 模型。

## 六、主要结论与发现

1. **线性层确为性能瓶颈**：其任务特异性不足限制了 MIL 模型上限。
2. **MAMMOTH 显著提升性能**：
   - 在大部分任务上优于复杂 MIL 聚合网络；
   - 无需改变总体参数量。
3. **形态学专门化（Morphological Specialization）**：
   - 专家在路由机制下能自动学到肿瘤区、基质区、淋巴细胞等不同表型。
4. **梯度冲突（Instance Gradient Interference, IGI）缓解**：
   - 不同专家可隔离异质性图块的梯度方向，稳定训练。
5. **更高数据效率**：
   - 即使训练数据稀缺时，MAMMOTH 仍较其他 MoE 方法稳健。
